はじめに
=============================================================

コンテナ型仮想化とサーバ仮想化の違いは、対象がマシンではなくプロセスになる。
サーバ仮想化ではホストマシン上でゲストマシンが動き、ゲストマシン内で様々なプロセスが動く。コンテナ型仮想化では、ホスト OS 上にコンテナという隔離された空間が作成され、その中でプロセスが実行される。

コンテナ内で動いているのはマシンではなく単なるプロセスであるため、当然ながら cron や syslog 等の他のプロセスは動いていない。「コンテナという隔離された空間で一つのプロセスが動く」という概念を持つこと。

1. セットアップ手順
   =============================================================

1. Docker Tool Box を `Docker のサイト <https://www.docker.com/products/docker-toolbox>`_ からダウンロード

アプリケーションフォルダに、以下の２つのアイコンが表示される。

* Kitematic（GUIツール）

  Dockerコンテナを一覧したり、設定を調整したりできるGUIツール。
    Downloadingocker Hubからイメージを追加したりできる。

    * Docker Quickstart Terminal

      クリックすると、Docker用ホストを起動し、Mac上から操作するための環境変数などを設定した上で、ターミナル(iTerm)が起動する。

      docker の環境設定として以下を bash_profile に追加する ::

        $ eval $(docker-machine env default)
